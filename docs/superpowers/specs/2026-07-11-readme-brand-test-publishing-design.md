# README、品牌图标与测试发布设计

日期：2026-07-11  
状态：已确认

## 目标

重写 NPlan 的双语 README，使新用户能快速理解、安装并使用产品；新增一个可直接用于 GitHub 和文档的矢量图标；从仓库当前版本移除测试源码并防止后续误上传。

本次仅调整文档、品牌资产和仓库发布边界，不改变 NPlan 的规划逻辑、CLI 行为或 planning-only 产品边界。

## README 设计

`README.md` 保持英文主文档，`README.zh-CN.md` 作为内容等价的中文版本。两份文档使用相同章节顺序：

1. 图标、项目名称和语言切换
2. 一句话产品定位
3. planning-only 边界
4. 30 秒快速开始
5. 工作流程
6. 核心能力
7. 常用命令与典型示例
8. 模型配置、安全与隐私
9. 进阶文档入口和许可证

README 采用“精简产品首页”结构。删除重复说明、长篇背景和过细的内部目录介绍；完整契约、模型配置和代理模块说明继续由 `docs/` 下的聚焦文档承载。

文案必须：

- 准确描述当前 v0.2 功能，不承诺任务执行、文件修改或远程代理能力。
- 以最短可运行路径作为快速开始，不要求用户先理解内部架构。
- 明确云模型上下文发送需要确认，本地模型不需要云端授权。
- 保持英文和中文的事实、命令、链接一致。

## 图标设计

新增 `assets/nplan-icon.svg`，采用已选定的“路径 N”方案：

- 512×512 SVG 画布。
- 深蓝色圆角方形背景。
- 蓝色到青色渐变的 N 形路径。
- 三个圆形节点分别表达理解、规划和校验。
- 线条、节点和留白需保证 32 px 尺寸仍可辨认。
- README 顶部引用该 SVG；不增加字标、PNG 或多套主题变体。

图标只表达规划过程，不使用机器人、终端、闪电或执行箭头等可能暗示自动执行的元素。

## 测试发布规则

测试源码保留在当前工作区，但不再包含在 Git 当前版本中：

- 将根目录 `/test/` 加入 `.gitignore`。
- 使用仅取消跟踪的 Git 操作移除已跟踪测试，不能删除本地文件。
- 不改写 Git 历史，不强制推送；旧提交中的测试仍然存在。
- 移除 `package.json` 的公开 `test` 脚本，避免新克隆仓库暴露一个缺少测试源码的入口。
- 维护者在拥有本地测试目录时，可直接运行 `node --test`，由 Node.js 自动发现测试文件。
- README 与当前聚焦文档不得把 `node --test` 表述为公共仓库开箱可用；出现该命令时必须明确依赖维护者私有的本地 `test/` 目录。

## 验证与发布

实施完成后必须确认：

- 两份 README 的章节、命令、链接和边界说明一致。
- README 中引用的本地文件均存在。
- SVG 是有效 XML，并在大尺寸和 32 px 预览下清晰。
- `package.json` 是有效 JSON，且不再包含 `scripts.test`。
- `docs/agent-module-spec.md` 和 `docs/local-knowledge.md` 都将 `node --test` 明确限定为维护者私有本地测试目录可用。
- `git ls-files test` 无输出，而本地 `test/` 目录及文件仍然存在。
- `.gitignore` 能匹配根目录测试目录。
- 工作树只包含本设计范围内的变更。

验证通过后提交到 `main` 并推送到现有 GitHub 远端。推送使用普通快进更新，不改写远端历史。

## 验收标准

- GitHub 项目首页能在短时间内说明 NPlan 是什么、如何开始以及不会做什么。
- 中英文用户获得等价信息，中文表达符合常见操作顺序。
- 图标与 planning-only 定位一致，并可直接用于 README。
- GitHub 当前版本不再包含测试源码，后续新增测试文件也不会被默认跟踪。
- 本地测试源码完整保留。
